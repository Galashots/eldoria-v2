## 2026-07-21 — Mage sprite scale consistency + despeckle (real-iPad play feedback)

- Author/branch: Claude Code (repo agent), `claude/mage-sprite-uniform-scale`.
- Scope: fixes two defects the repo owner observed on a physical-iPad playtest — the Mage changing size by facing direction, and stray dots around the sprite. Root causes (measured): (1) the normalizer's per-frame trim+contain scaling let each direction row's trim extent set its own scale (idle left rendered ~23% smaller than idle right; idle→walk also jumped); (2) idle/walk source rows bleed across grid-cell boundaries, so blind cell slicing gave left-row cells a ghost hair fragment (the "dots" below the sprite) and clipped the right row's hair; (3) semi-alpha AI-generation residue survived background removal. Normalizer (`scripts/normalize-asset-sheet.mjs`) gains `fit: "fixed"` with a required per-frame `scale`, and per-source `cleanup: { alphaFloor, minComponentPx }` (rejected for `edge_flood_color_key` sources, where the background is still opaque when cleanup runs). All four mage manifests were rewritten to per-sprite `sourceRect`s (padded cleaned component-union bounds), `cleanup: { alphaFloor: 64, minComponentPx: 30 }` (preserves the smallest intentional cast sparkle at 33px and hurt pain stars at 392+px), and per-clip fixed scales sharing one 46.7px body height across every idle/walk direction; cast/hurt use their largest feasible per-clip scales (cast bodies are up to ~10% smaller only while the wide staff-and-orb pose occupies the cell). Runtime masters regenerated deterministically from committed sources + manifests.
- Verification: `npm run check` exit 0; `npm run test:asset-pipeline` passes with new red/green coverage (cleanup removes crumbs/fringe and tightens trim bounds — with the fixture proving the dirty path differs; fixed-fit renders identical body size across frames with different trim extents; validator rejects missing/misplaced/negative `scale`, bad cleanup values, and over-cell fixed scales at normalize time); `npm run test:unit` 150/150; full smoke suite result recorded in the PR. Measured post-fix sheets: idle/walk 20–22×47–48 in all directions with 0–1 stray pixels per frame (was one row ~23% smaller with up to 8 strays). Before/after sheet comparisons and in-game four-direction captures: `docs/playtests/2026-07-21-mage-sprite-fix/`.
- Compatibility: no save, quest, curriculum, profile-ID, dependency, or code-runtime changes — texture keys, frame counts, cell geometry, and animation configs are unchanged; only the sheet pixels and the pipeline that produces them changed. Slime/tile manifests are untouched (default `contain` behavior preserved byte-for-byte by the new code paths).
- Remaining risk: exact runtime pixels changed, which re-opens the ChatGPT visual-audit gate for the mage sheets; cast/hurt body sizes are limit-capped below the idle/walk baseline (brief scenes, uniform within each clip); physical-iPad re-check of the fixed sprite is pending.
